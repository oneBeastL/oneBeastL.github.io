Github Pages创建自己的独立博客

作为开源代码库以及版本控制系统, Github拥有140多万开发者用户. 随着越来越多的应用程序转移到了云上, Github已经成为了管理软件开发以及发现已有代码的首选方法. Github作为现在最流行的代码仓库，已经得到很多大公司和项目的青睐，比如jQuery、Twitter等.
为了使项目更方便的被人理解, Github替你想到了这一点, 他提供了Github Pages的服务, 不仅可以方便的为项目建立简介站点, 同时也可以建立个人博客.

Github Pages有以下几个优点:

  • 轻量级的博客系统, 没有配置的麻烦
  • 可以使用标记语言, 如Markdown
  • 无需自己搭建服务器
  • 根据Github的限制, 对应每个账号下的站有300M的空间
  • 可以绑定自己的域名

当然他也有缺点:

  • 使用jekyll模板系统, 静态页发布, 适合博客, 文档介绍等
  • 动态程序部分比较局限, 比如没有评论, 我们可以使用其他方法进行设置
  • 基于Git, 需要动手能力强, 不像Wordpress有强大的后台

作为个人博客来说, 清爽的界面搭配, 简单的操作, 就能够表达自己的工作、学习心得, Github Pages是一个很好的方案.

Github帐号配置

注册帐号

首先， 我们需要在Github上面创建一个账号。如果你还没有github账号，那么可以在首页进行快速注册

如果没有看懂, 可以点击教程进行注册

注册好帐号之后, 我们点击下图中的按钮创建博客仓库。

我们点击上图的+号创建新项目，项目名称必须为: github名字.github.io，比如本文中的github名称为oneBeastL，那么新仓库的名字必须为: oneBeastL.github.io

设置完之后, 我们点击创建(上图黑色剪头所指).我们完成了博客仓库的创建, 接下来要进行配置啦

Hexo环境搭建

根据电脑操作系统(mac、windows)下载对应版本:

1. Node.js

用来生成静态页面. 移步Node.js官网, 下载最新版Stable安装即可.

  • 下载 Node.js

2. Git

用来将本地Hexo内容提交到Github上。Xcode自带Git，这里不再赘述。如果没有Xcode可以参考Hexo官网上的安装方法。

  • 下载 Git

Hexo

安装好Git和Node.js之后, 在终端下输入下面这条命令来安装Hexo:

npm install hexo-cli -g

稍微等待一会就会出现安装信息

我们来测试一下, 是否安装成功, 输入下面的命令:
hexo -v

如果出现类似内容说明安装成功啦!

Hexo的使用

很好! 现在我们开始创建博客写文章, 下面会介绍几个hexo常用的指令.

新建博客文件夹
我们在桌面创建个名字为blog的文件夹(按照自己的喜好该文件夹可以放在任意位置, 当然名字也是可以随意的). 现在我们用终端进入到这个文件夹里, 然后将它初始化为我们的博客目录:
hexo init

注意看第二条的提示, 提示我们要执行npm install, 那么我们就执行这条指令
npm install

生成页面
写好文章之后怎么才能看到效果呢? 只需要下面的一条命令就能将我们Markdown文件(.md后缀)转换成.html

页面预览
我们需要在本地生成一个小型服务器, 把博客挂载上去, 才能访问. 只需要下面一条指令:

hexo server

我们会看到这样一条提示:

打开浏览器访问http://localhost:4000/.

如果欣赏完毕, 我们按照提示Ctrl+C退出小型服务器

配置信息
我们找到博客根目录下的配置文件_config.yml, 用文本编译器或者是Markdown编辑器打开它, 进行修改配置, 达到我们想要的样子. 可以查看Hexo官方文档对照修改
